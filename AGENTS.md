# AGENTS.md — AI 协作开发规则

> 本文件是 AI 编码助手的工作规范，修改代码前必须先阅读本文。

---

## 项目目标

开发一个个人博客网站。具体功能以 `docs/PRD.md` 为准。

## 技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Vite | 8.x | 构建工具 |
| React | 19.x | UI 框架 |
| TypeScript | 5.x | 类型安全 |
| Tailwind CSS | 4.x | 样式 |
| react-markdown | 10.x | Markdown 渲染（文章阅读） |
| remark-gfm | 4.x | GFM 扩展（表格、删除线等）|
| react-router-dom | 7.x | 路由 |
| localStorage | - | 数据持久化 |
| pnpm | - | 包管理器 |

## 开发原则

1. **优先保持简单，不要过度设计。**
2. **每次只实现一个明确的小功能。**
3. **不要一次性重构整个项目。**
4. **修改代码前，先说明计划，等确认后再动手。**
5. **修改完成后，说明修改了哪些文件。**
6. **每次修改后必须运行构建命令，确保项目可正常构建。**
7. **如果构建失败，先修复失败原因，再继续。**

## 文件结构约定

- `src/components`：通用 UI 组件
- `src/pages`：页面组件
- `src/types`：TypeScript 类型定义
- `src/utils`：工具函数
- `src/hooks`：自定义 React Hooks
- `src/storage`：本地存储相关逻辑
- `src/styles`：样式文件
- `src/assets`：静态资源
- `docs`：项目文档

## 禁止事项

1. 不要引入大型状态管理库（如 Redux），除非明确要求。
2. 不要添加后端服务或数据库，除非明确要求。
3. 不要添加登录注册功能，除非明确要求。
4. 不要删除现有功能。
5. 不要静默忽略 TypeScript 错误。
6. 不要在未经确认的情况下重构架构。
7. 不要把所有代码堆在一个文件里。

## 工作节奏

```
需求 → 计划（等确认）→ 实现 → 构建验证 → 人工验收 → 提交 → 下一个需求
```

## 构建命令

```bash
pnpm install    # 安装依赖
pnpm dev        # 启动开发服务器 → http://localhost:5173
pnpm build      # 构建生产版本 → dist/
pnpm preview    # 预览生产版本
```

## 验收检查清单

每阶段完成后必须检查：
1. 项目能否启动
2. 页面能否打开
3. 浏览器控制台无红色错误
4. 核心功能是否可操作
5. 构建是否通过
