# PRD.md — 产品需求文档

## 1. 项目名称

**玄牙个人世界**

## 2. 项目目标

> 个人写作博客 Web 应用。

- **是什么**：一个支持 Markdown 写作的个人博客，有发布/草稿状态、导出备份、文章标识
- **解决什么**：需要一个属于自己的写作空间，沉淀精神和知识
- **谁用**：仅祝欣缘（玄牙）自己使用
- **第一版范畴**：文章列表 + 发布/草稿状态 + 文章详情 + Markdown 写作/编辑/删除 + 导出备份 + 文章 emoji 标识 + 本地存储

## 3. 目标用户

- 我自己（玄牙）
- 单用户，不考虑多人

## 4. 核心功能（MVP 第一版）

- [x] 文章列表页 — 按时间倒序展示已发布文章，含标题、摘要、emoji、日期
- [x] 文章详情页 — 阅读模式，Markdown 渲染为富文本
- [x] 新建文章 — Markdown 编辑器（textarea + 实时预览分栏）
- [x] 编辑文章 — 修改已有文章内容
- [x] 删除文章 — 删除确认后移除
- [x] 发布/草稿状态 — 文章有 `draft` / `published` 两种状态，首页只显示已发布
- [x] 导出备份 — 一键导出全部文章为 JSON 文件，下载到本地
- [x] 本地持久化 — 数据保存到浏览器 localStorage，刷新不丢失
- [x] 文章 emoji 标识 — 写作时自选 emoji，列表页显示作为视觉区分
- [ ] 学术风 UI + 杂志美感 — 无衬线主字体、宽行距、大留白、讲究的排版

## 5. 页面结构

```
/               → 首页（仅展示已发布文章列表）
/write          → 新建文章（Markdown 编辑器 + 实时预览）
/write/:id      → 编辑文章
/article/:id    → 文章详情（阅读模式）
```

## 6. 数据结构

```typescript
interface Article {
  id: string;          // 唯一标识，时间戳生成
  title: string;       // 文章标题
  content: string;     // Markdown 原始内容
  emoji: string;       // 文章标识 emoji，默认 📝
  status: 'draft' | 'published';  // 草稿 / 已发布
  createdAt: number;   // 创建时间戳
  updatedAt: number;   // 最后修改时间戳
}
```

## 8. 界面风格：学术风 + 杂志美感

| 要素 | 方案 |
|---|---|
| 中文字体 | 系统无衬线（PingFang / Microsoft YaHei / Noto Sans SC） |
| 正文字号 | 18px，宽行距 1.8 |
| 内容宽度 | 阅读页 700px 居中，最大化留白 |
| 标题样式 | 大号、bold、有区分度的层级 |
| 排版 | 两端对齐，段落间距舒适 |
| 配色 | 中性灰白调为主，点缀低饱和度蓝/绿 |
| 整体感受 | 像一篇排版精致的 PDF 论文，但不压抑 |

## 9. Markdown 编辑器方案

**方案 A：纯 textarea + 右侧实时预览分栏**

- 左侧：textarea，输入 Markdown 原文
- 右侧：react-markdown 实时渲染预览
- 全屏宽的分栏布局
- 无额外编辑器依赖

## 10. 技术栈

| 技术 | 用途 |
|---|---|

| Tailwind CSS | 样式 |

| localStorage | 数据持久化 |
| react-router-dom | 路由 |
