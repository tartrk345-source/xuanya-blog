# 开发流程.md — Vibe Coding 开发流程

> 本文件记录了本项目采用的完整 Vibe Coding 开发方法。
> 每次开发新功能前，先回顾本文。

---

## 核心原则

| 原则 | 说明 |
|------|------|
| 人做决策 | 你负责定义目标、确认计划、验收结果 |
| AI 做执行 | AI 负责写代码、运行命令、修复错误 |
| 一次一步 | 每个阶段只做一个明确的小功能 |
| 每步可运行 | 任何阶段结束后，项目必须能正常启动和构建 |
| 先计划后动手 | AI 修改代码前必须先说明计划并等你确认 |

---

## 完整流程 12 步

### 第 1 步：明确产品目标
写清楚四个问题：
- 这个软件是什么？
- 它解决什么问题？
- 谁用？
- 第一版做到什么程度？不做哪些？

输出：一段简短的文字描述

### 第 2 步：生成 PRD（MVP 需求文档）
让 AI 整理需求文档，包含：
- 项目目标
- 核心功能
- 页面结构
- 数据结构
- 不做的功能
- 开发阶段划分

输出：`docs/PRD.md`

### 第 3 步：选择技术栈
新手优先：Vite + React + TypeScript + Tailwind CSS + localStorage
（如需登录/数据库，改用 Next.js + Supabase）

### 第 4 步：初始化项目
创建空项目，确保能运行和构建

### 第 5 步：建立项目规则文档
确保 `AGENTS.md` 存在且内容完整

### 第 6 步：制定分阶段开发计划
让 AI 阅读项目，输出分阶段计划（先不写代码，等确认）

### 第 7 步：分阶段实现
一次只做一个阶段，每阶段约 2-4 个小功能

### 第 8 步：每阶段验收
5 项检查：
1. 项目能启动
2. 页面能打开
3. 无控制台红色错误
4. 核心功能可操作
5. 构建通过

### 第 9 步：Git 保存版本
验收通过 → 提交 commit（commit message 用中文）

### 第 10 步：问题修复
反馈时说明：操作步骤 + 实际结果 + 期望结果 + 错误信息

### 第 11 步：代码审查
MVP 完成后的代码体检，先只出报告

### 第 12 步：部署上线
Vercel / Netlify / Cloudflare Pages

---

## 工作节奏

| 天数 | 任务 |
|------|------|
| 第 1 天 | 写想法 → PRD → 选技术栈 → 初始化 → 建立规则 → 空项目跑通 |
| 第 2 天 | 静态版本：布局 + 假数据 + 样式 + 移动端适配 |
| 第 3 天 | 核心交互：增删改查 + 表单 + 空状态 |
| 第 4 天 | 数据保存：持久化 + 统计 + 异常处理 |
| 第 5 天 | 整理部署：审查 + 修 bug + README + 部署 |

## AI 协作常用提示词模板

### 让 AI 读项目并出计划
```
请阅读当前项目结构、AGENTS.md 和 PRD.md。
然后输出分阶段开发计划：
1. 分成多个小阶段
2. 每个阶段有明确功能和验收标准
3. 先不要修改代码
```

### 让 AI 实现一个阶段
```
请实现阶段 [编号]：[名称]。
要求：
1. 修改前先说明计划
2. 只做这个阶段的内容
3. 修改后运行构建命令
4. 说明修改了哪些文件
```

### 让 AI 修复 bug
```
当前问题：
操作步骤：[1.xxx 2.xxx 3.xxx]
实际结果：[xxx]
期望结果：[xxx]
错误信息：[粘贴]
请修复。要求：先分析原因，只修改必要文件。
```
